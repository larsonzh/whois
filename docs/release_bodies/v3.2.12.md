# whois v3.2.12

本次版本聚焦 Step 4.7 工程化收口与发布前门禁流程固化，保持默认查询语义不变（开关默认关闭）。
This release focuses on Step 4.7 engineering closure and prerelease gate solidification while preserving default query semantics (flags remain OFF by default).

中文概要：
- Step 4.7 工程化收口：完成受控 trial/early-unknown 能力链路，并新增 `--step47-early-unknown-list <csv>` 候选可配置入口。
- 默认语义保持不变：Step 4.7 相关能力均为受控开关，默认关闭，不改变默认查询路径与输出契约。
- 一键门禁落地：新增 `tools/test/step47_prerelease_check.ps1`，串联 readiness + A/B + rollback 并输出统一汇总。
- 任务面板入口：新增 VS Code 任务 `Test: Step47 PreRelease Check (reserved, list file)`，复用 `step47ListFile` 输入，降低手工参数漂移。
- 脚本维护性：`tools/test/step47_rollback_drill.ps1` 函数命名统一为 approved verbs（`ConvertTo-*` / `Get-*`），不改变断言语义。
- 验证基线（2026-03-16）：
  - Step47 一键门禁 PASS：`out/artifacts/step47_prerelease/20260316-043150`
  - Remote Strict PASS：`out/artifacts/20260316-043644`（`Local hash verify PASS` + `Golden PASS` + `referral check PASS`）
  - CIDR Contract Bundle PASS：`out/artifacts/cidr_bundle/cidr_bundle_summary_20260316-043717.txt`（body `pass=4 fail=0`，matrix `pass=9 fail=0`）
  - Redirect Matrix 10x6 PASS：`out/artifacts/redirect_matrix_10x6/20260316-043753`（`authMismatchFiles=0`，`errorFiles=0`）

English summary:
- Step 4.7 engineering closure: complete the controlled trial/early-unknown capability chain, and add configurable candidates via `--step47-early-unknown-list <csv>`.
- Default semantics unchanged: Step 4.7 capabilities are controlled flags and remain OFF by default, preserving default query flow and output contracts.
- One-command gate: add `tools/test/step47_prerelease_check.ps1` to chain readiness + A/B + rollback with unified summaries.
- Task-panel entry: add VS Code task `Test: Step47 PreRelease Check (reserved, list file)`, reusing `step47ListFile` to reduce manual parameter drift.
- Script maintainability: normalize function names in `tools/test/step47_rollback_drill.ps1` to approved verbs (`ConvertTo-*` / `Get-*`) without assertion semantic changes.
